A composition of continuous functions is continuous: statement and full proof, then the pieces.

Click the dotted pieces; read in order the first time.

On paper: given \(\varepsilon\), continuity of \(g\) at \(f(a)\) yields \(\delta_1\); continuity of \(f\) at \(a\), applied with \(\delta_1\) in the role of \(\varepsilon\), yields \(\delta_2\); and \(\delta_2\) works. The Lean proof is those three sentences, one line each.

theorem — a def whose type is a proposition

The shape is identical to a def: name, arguments, colon, type, :=, term of that type. Here the type is a proposition, so the body is a proof. One consequence: Lean never inspects a finished proof again (proof irrelevance) — two proofs of the same proposition are interchangeable, as in practice, where one cites a theorem and not its proof.

{f g : ℝ → ℝ} {a : ℝ} — implicit arguments

Braces instead of parentheses mark arguments the user will not supply: Lean infers them. Given hf : ContinuousAt' f a, both f and a can be read off the type of hf. This is the formal counterpart of context in written mathematics — after “by continuity of \(g\) at \(f(a)\)…” one never repeats what \(g\) is.

(hf : ContinuousAt' f a) — hypotheses are arguments with names

A hypothesis is one more typed argument whose type is a proposition; hf is a proof of continuity, available for use in the body. Naming proofs is the one unfamiliar habit — on paper hypotheses sit anonymously in the preamble; here they are referred to later, so they need handles (hf, “the hypothesis on \(f\)”, is the Mathlib convention). Note the second hypothesis is continuity of \(g\) at the point \(f(a)\), exactly what the argument requires.

ContinuousAt' (g ∘ f) a — the goal

What follows the final colon is the statement proved — the type of the theorem. ∘ is composition, defined by (g ∘ f) x = g (f x). Lean unfolds such definitions silently when checking; equality that holds by unfolding definitions is called definitional, and it is why the last proof line is accepted without an explicit rewriting step. (The distinction between definitional and proved equality returns as a cost in Part 2.)

:= by — tactic mode

After by, one writes commands (tactics) that build the proof step by step; at each point Lean displays the goal state — current hypotheses and what remains. The panels below reproduce it. This loop is the actual experience of using Lean, and the editor at the bottom of the page runs it on this very proof. Tactics are a convenience layer, not a second logic: the script produces an ordinary term, shown after the notes. (Systematic treatment: ch. 5 of Theorem Proving in Lean 4.)

intro ε hε — “let \(\varepsilon > 0\) be given”

The goal unfolds to ∀ ε > 0, ∃ δ > 0, …; intro ε hε fixes an arbitrary \(\varepsilon\) and a proof hε of its positivity. Two names because, without abbreviations, there are two things: a quantifier and an implication (previous page).

goal after intro ε hε f g : ℝ → ℝ
a : ℝ
hf : ContinuousAt' f a
hg : ContinuousAt' g (f a)
ε : ℝ
hε : ε > 0
⊢ ∃ δ > 0, ∀ (x : ℝ), |x - a| < δ → |(g ∘ f) x - (g ∘ f) a| < ε

obtain ⟨δ₁, hδ₁, hg'⟩ := hg ε hε — two ideas at once

Applying a proof. hg proves a universally quantified statement, hence is a function; hg ε hε — applied to the number and to the positivity proof — proves the instantiated body ∃ δ > 0, ∀ x, |x - f a| < δ → |g x - g (f a)| < ε. “Apply continuity of \(g\) with this \(\varepsilon\)” is function application.

Opening a pair. A proof of an existential is a witness with evidence; obtain ⟨δ₁, hδ₁, hg'⟩ := … names the parts. This is the formal “choose \(\delta_1\) that works”.

goal after the two obtain lines δ₁ : ℝ
hδ₁ : δ₁ > 0
hg' : ∀ (x : ℝ), |x - f a| < δ₁ → |g x - g (f a)| < ε
δ₂ : ℝ
hδ₂ : δ₂ > 0
hf' : ∀ (x : ℝ), |x - a| < δ₂ → |f x - f a| < δ₁
⊢ ∃ δ > 0, ∀ (x : ℝ), |x - a| < δ → |(g ∘ f) x - (g ∘ f) a| < ε

The second obtain applies hf not with \(\varepsilon\) but with \(\delta_1\) — the \(\delta\) of one function in the role of the \(\varepsilon\) of the other. That substitution is the mathematical content of the theorem, and it is visible in the script as which argument is passed where.

exact — supplying the required object

exact t closes the goal by giving a term t of exactly the goal’s type — the escape from tactics back to direct writing, used when one can simply state the answer.

⟨δ₂, hδ₂, …⟩ — building the pair

The same brackets that obtain used to open an existential here build one: witness \(\delta_2\), positivity, main property. The symmetry is worth noting — \(\exists\) is used by taking apart and proved by assembling, while \(\forall\)/\(\to\) are used by applying and proved by the function of the next note.

fun x hx => hg' (f x) (hf' x hx) — the proof is a function

The remaining component must prove ∀ x, |x - a| < δ₂ → |(g ∘ f) x - (g ∘ f) a| < ε, so we write the function: given \(x\) and hx : |x - a| < δ₂,

hf' x hx        : |f x - f a| < δ₁
hg' (f x) (…)   : |g (f x) - g (f a)| < ε

Evidence about \(x\) is processed by hf'; its output is precisely the input hg' requires. The composite estimate is the composite of the two evidence-processing functions. (That the goal says (g ∘ f) x while the term produces g (f x) is definitional equality — the goal note.)

The same proof with no tactics

The script above is an interactive way of writing a term; the term itself:

One expression: a function that takes \(\varepsilon\) with its positivity proof, opens two pairs, and assembles a third. Checking the proof means type-checking this term. In practice one writes tactics for anything nontrivial, but they compile to this, which is all the type checker ever sees.

Exercise

The editor contains the theorem — step through it and watch the goal panel — and an exercise: if \(f\) is continuous at \(a\), so is \(x \mapsto f(x) + c\), with the same \(\delta\). The skeleton is set up; sorry (Lean’s placeholder for a missing proof) is yours to replace. After have h := hf' x hx, the goal differs from h by the identity \((f(x)+c) - (f(a)+c) = f(x) - f(a)\); the tactic convert h using 2 reduces the goal to that identity, and ring closes it.

For more exercises of this kind: the Natural Number Game (Buzzard–Eugster) and Mathematics in Lean (Avigad–Massot), which continues from roughly this point at book length.
