Notes on the proof assistant Lean 4, written by a student, for mathematicians. They run in the opposite direction from the standard texts: instead of building up from the foundations, they start from a theorem and take it apart.

The standard introductions are bottom-up. Theorem Proving in Lean 4 opens with a chapter on dependent type theory; the Natural Number Game constructs \(\mathbb{N}\) from the Peano axioms; even Mathematics in Lean, the standard mathematician-oriented text, begins with equalities proved by calculation. Here the order is reversed: Part 1 reads a complete definition and a complete proof first — the \(\varepsilon\)-\(\delta\) definition of continuity, and the theorem that a composition of continuous functions is continuous — and recovers the grammar from them. The idea of showing a finished analysis proof before any syntax comes from the opening file of Patrick Massot’s Glimpse of Lean.

Part 1 · Anatomy of a theorem

The definition of continuity and the composition theorem, read piece by piece: binders, quantifiers, tactics, and in what sense a proof is a function.

Part 2 · Why dependent types?

One statement written in set theory, in simple type theory, and in dependent type theory; the case for the third, its costs, and its limits.

Prerequisites and format. A mathematics degree in progress or the equivalent; no programming background. Nothing needs installing: the code boxes marked as exercises load the Lean web editor in the page. In Lean code, dotted-underlined pieces are clickable and open the explanation beneath them. Read Part 1 in order; Part 2 is independent if you already know some Lean.

The single fact everything else depends on: in Lean, a statement is a type, a proof is an element of that type, and checking a proof is checking that the element has the type.
