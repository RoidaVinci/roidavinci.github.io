The statement “\(f\) is continuous at \(a\)” — for every \(\varepsilon > 0\) there is \(\delta > 0\) such that \(|x - a| < \delta\) implies \(|f(x) - f(a)| < \varepsilon\) — written in Lean, then read piece by piece.

Each dotted piece is clickable and opens its explanation below. Read in order the first time.

def — a Lean file is a sequence of declarations

def introduces a named object. A Lean development, like a paper, is a sequence of declarations; the four common kinds are def (an object), theorem (a proof of a stated proposition), example (an anonymous theorem, checked and discarded), and structure (a bundled package such as “a set together with an operation”). That def and theorem are nearly the same construction is explained at : Prop.

(f : ℝ → ℝ) (a : ℝ) — the colon reads “is a”

These are the arguments: \(f \colon \mathbb{R} \to \mathbb{R}\) and \(a \in \mathbb{R}\). In Lean, x : T asserts that x has type T, and this is not a proposition to be proved — it is a grammatical fact, checked mechanically. Where set theory writes \(a \in \mathbb{R}\) and treats it as a statement, Lean treats a : ℝ like a part of speech: an expression that fails the check is not false but ill-formed.

Functions are primitive

The type ℝ → ℝ is a primitive notion: a function is a rule that can be applied, not a set of ordered pairs. The set-of-pairs definition is a foundational encoding; type theory takes the working concept as basic.

Arguments are taken one at a time

ContinuousAt consumes f and returns a function of a. Hence ContinuousAt (fun x => x^2) alone is meaningful, of type ℝ → Prop: the predicate “\(x^2\) is continuous at —”. Partial application of this kind (currying) is used constantly.

: Prop — a statement is a type

The definition’s result type is Prop, the type of propositions. Compare: 3 : ℝ and ℝ : Type, while ContinuousAt f a : Prop. The central point: ContinuousAt f a is itself a type, and its elements are the proofs of continuity — to prove the proposition is to exhibit an element, h : ContinuousAt f a. This is the propositions-as-types principle (Curry–Howard). It explains why def and theorem coincide (a theorem defines an element of a proposition), and why a single small type checker suffices to verify proofs. Kevin Buzzard’s Mathematics in type theory develops this picture in a few pages.

The tower above

Types have types: ℝ : Type, Type : Type 1, Type 1 : Type 2, … This hierarchy of universes is Lean’s answer to Russell’s paradox, analogous to the set/proper-class distinction. It returns as a genuine limitation in Part 2.

:= — “is defined to be”

:= for definitions; plain = for equations one asserts or proves. A def is an abbreviation, exactly as in ordinary mathematics: Lean unfolds it silently whenever needed, no axiom is added, and nothing is proved here. Whether any function satisfies the definition is the next page’s question.

∀ ε > 0, — quantifiers belong to the grammar

Two conventions to know. First, ∀ ε > 0, P ε abbreviates ∀ ε, ε > 0 → P ε — a quantifier over all of \(\mathbb{R}\) followed by an implication, as in the usual reading “for every \(\varepsilon\), if \(\varepsilon > 0\) then…”. Second, the range of a quantifier may be omitted: we wrote ∀ x, with no type, and Lean infers \(x \in \mathbb{R}\) because x is later an argument of f. This inference (elaboration) is what keeps the formal text close to the written one.

What a proof of \(\forall\) is

A proof of ∀ x : ℝ, P x is a function assigning to each \(x_0\) a proof of \(P(x_0)\). Its type, written (x : ℝ) → P x, has the variable \(x\) occurring inside it — a dependent function type. Part 2 is about this construction.

∃ δ > 0, — existence

Likewise an abbreviation: ∃ δ > 0, P δ stands for ∃ δ, δ > 0 ∧ P δ — with a conjunction where the universal quantifier had an implication, the same asymmetry taught in a first logic course.

What a proof of \(\exists\) is

A pair: a witness together with a proof about that witness, written ⟨δ₀, h⟩. Since the second component’s type mentions the first, this is a dependent pair — the companion of the dependent function type above. Both are dismantled and rebuilt in the next page’s proof.

→ — implication is the function arrow

The same arrow appears in f : ℝ → ℝ and in |x - a| < δ → |f x - f a| < ε, and it is one symbol with one meaning. Propositions are types, so an implication \(P \to Q\) is the type of functions carrying proofs of \(P\) to proofs of \(Q\); “assume \(P\)… then \(Q\)” is function application. A proof that takes an \(\varepsilon\) and produces a \(\delta\) is, in Lean, literally a function doing so.

f x — application is juxtaposition

Lean writes f x for \(f(x)\); parentheses only group. So g (f x) is \(g(f(x))\), while g f x would mean \((g(f))(x)\) — f itself passed to g. The convention makes partial application cheap, and mathematics is full of it: “the derivative of \(f\) at \(a\) in direction \(v\)” is three applications in a row.

|x - a| < δ — notation is resolved by type, and extensible

The absolute value, subtraction, order, and the numeral 0 are not built in; they are notations resolved by the types involved. Because x and a are real, Lean selects the absolute value, order, and zero of \(\mathbb{R}\) — the same mechanism (type classes) later gives |·| its meaning on \(\mathbb{C}\) or any normed field, so a statement is written once at the right generality. All of Mathlib’s notation (∑ i ∈ s, f i, 𝓝 a, …) is user-defined in this way.

The definition without abbreviations

Nothing further is hidden: the definition is an alternation of dependent function types (\(\forall\), \(\to\)) and dependent pairs (\(\exists\), \(\wedge\)) over \(\mathbb{R}\). A proof of it will accordingly be a function carrying each \(\varepsilon\) with its positivity proof to a \(\delta\) bundled with evidence — the next page constructs one.

Mathlib has its own ContinuousAt, defined for maps of arbitrary topological spaces via filters (Tendsto f (𝓝 x) (𝓝 (f x))); for metric spaces, Metric.continuousAt_iff identifies it with the \(\varepsilon\)-\(\delta\) form. Part 1 keeps the \(\varepsilon\)-\(\delta\) definition so that every symbol on screen is one we defined.

Take \(\delta = \varepsilon\): a proof of the existential is a triple \(\langle\)witness, positivity, main property\(\rangle\), and here the main property can pass its hypothesis straight through — if stuck, try exact ⟨ε, hε, fun x hx => hx⟩.
